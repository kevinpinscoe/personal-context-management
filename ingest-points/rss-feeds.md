# Ingest Point: Custom RSS Feeds

## Summary

Custom RSS/XML feeds generated by Python scripts in `~/private-tools/` that scrape external websites and write structured feed data to `~/rss-feeds/`. These are **not** subscriptions to external RSS endpoints — each feed is built and maintained locally from scraped or fetched source data.

Full operational documentation lives in the `~/rss-feeds/` repo:

- `~/rss-feeds/README.md` — repo overview
- `~/rss-feeds/4chan/RUNBOOK.md`
- `~/rss-feeds/hacker-news/RUNBOOK.md`
- `~/rss-feeds/reddit/RUNBOOK.md`

### 4chan Boards

| Item | Detail |
|---|---|
| **Script** | `~/private-tools/4chan-tech-feed-converted.py` |
| **Source API** | `https://a.4cdn.org/{board}/2.json` (official 4chan API) |
| **Secrets** | None |
| **Automation** | systemd timer — 02:00 and 14:00 daily |

**Boards and output files:**

| Board | Feed file |
|---|---|
| `/g/` — Technology | `~/rss-feeds/4chan/tech/feed.xml` |
| `/sci/` — Science & Math | `~/rss-feeds/4chan/sci/feed.xml` |
| `/diy/` — Do It Yourself | `~/rss-feeds/4chan/diy/feed.xml` |

**Format:** RSS 2.0 XML. Each item is a thread OP. Items older than 30 days are dropped automatically by the script on each run.

**PCM ingest notes:**
- Read items directly from `feed.xml`; no SQLite database to update.
- Items age out of the feed naturally — no explicit deletion step required after processing.
- Strip HTML from `<description>` fields (thread OP text is embedded as raw HTML).

### Hacker News

| Item | Detail |
|---|---|
| **Script** | `~/private-tools/hn-2026-catchup.py` |
| **Source** | `https://news.ycombinator.com/front?day=YYYY-MM-DD` |
| **Secrets** | None |
| **Automation** | Manual / on-demand (run daily to stay current) |

**Files:**

| File | Purpose |
|---|---|
| `~/rss-feeds/hacker-news/2026/hacker-news-2026.xml` | RSS 2.0 feed of all 2026 HN front-page articles |
| `~/rss-feeds/hacker-news/2026/progress.db` | SQLite — tracks seen article IDs and fetch state |

**`progress.db` schema (relevant tables):**

```sql
-- Tracks every article indexed into the feed
CREATE TABLE articles (
    -- id, title, link, added_at, and related fields
    added_at TEXT,
    title    TEXT
    -- see RUNBOOK for full schema
);

-- Tracks the next date to fetch
CREATE TABLE state (
    key   TEXT PRIMARY KEY,  -- 'next_date'
    value TEXT
);
```

**PCM ingest requirement — delete processed articles from `progress.db`:**

```python
import sqlite3
conn = sqlite3.connect("/home/kinscoe/rss-feeds/hacker-news/2026/progress.db")
conn.execute("DELETE FROM articles WHERE link = ?", (article_url,))
conn.commit()
conn.close()
```

Also remove the corresponding `<item>` from `hacker-news-2026.xml` after processing, or leave for the next scrape cycle to age out.

### Reddit Saved Posts

| Item | Detail |
|---|---|
| **Script** | `~/private-tools/run-reddit-saved` → `sanitize-reddit-saved-feed.py` |
| **Source** | Reddit private saved-items Atom feed (up to 100 items per fetch) |
| **Secrets** | OpenBao `app/reddit` → field `saved_rss_url` |
| **Automation** | Manual / on-demand |

**Files (organized by calendar month):**

| File | Purpose |
|---|---|
| `~/rss-feeds/reddit/seen.db` | SQLite — deduplication; every ingested item tracked here |
| `~/rss-feeds/reddit/YYYY-MM/saved-links.rss` | Cumulative RSS 2.0 for that calendar month |
| `~/rss-feeds/reddit/YYYY-MM/reddit-saved-sanitized.json` | New items from the most recent run (JSON) |
| `~/rss-feeds/reddit/YYYY-MM/reddit-saved-sanitized.md` | New items from the most recent run (Markdown) |

**`seen.db` schema:**

```sql
CREATE TABLE seen_items (
    guid        TEXT PRIMARY KEY,  -- cleaned Reddit post URL
    title       TEXT,
    link        TEXT,
    ingested_at TEXT NOT NULL,     -- ISO 8601 UTC
    month       TEXT NOT NULL      -- YYYY-MM of monthly subdir
);
```

**PCM ingest requirement — delete processed items from `seen.db`:**

```python
import sqlite3
conn = sqlite3.connect("/home/kinscoe/rss-feeds/reddit/seen.db")
conn.execute("DELETE FROM seen_items WHERE guid = ?", (item_guid,))
conn.commit()
conn.close()
```

Deleting a row allows the item to be re-ingested on the next Reddit feed fetch if it is still in the Reddit saved feed. If the intent is to permanently retire the item, unsave it from Reddit directly (see the future purge job described in the Reddit RUNBOOK).

## General PCM Pipeline Notes

- All scripts are **Python 3.13, stdlib only** — no virtualenv required beyond `mise install` in `~/private-tools/`.
- Feed data files (XML, SQLite) are **gitignored** in `~/rss-feeds/` — local disk only.
- Process items from the JSON or Markdown sidecars where available (Reddit) — they are pre-sanitized. For HN and 4chan, parse the XML directly.
- **Always delete the processed item's SQLite row** (HN: `articles`, Reddit: `seen_items`) after a successful Obsidian note is written. This prevents reprocessing on the next pipeline run.
